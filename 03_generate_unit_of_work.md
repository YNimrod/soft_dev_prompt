# Phase 3: Unit of Work（作業単位）分割

## プロンプト

```
## Unit of Work 定義

あなたの役割: あなたは経験豊富なソフトウェアアーキテクトです。
タスクセクションにあるシステム全体のユーザーストーリーを理解する任務を担っています。

並行して実装できる複数の作業ユニットにストーリーをグループ化してください。
各ユニットには、単一のチームで構築できる高い凝集性を持つユーザーストーリーが含まれます。
ユニットはドメイン駆動設計における「境界づけられたコンテキスト」に相当し、
特定のサブドメインまたは特定のビジネスロールに沿ったものです。

各ユニットについて、それぞれのユーザーストーリーと受け入れ基準を
{{ドキュメントルート}}/inception/units/ フォルダの個別の md ファイルに記載してください。
追加の設計詳細は生成しないでください。

作業の計画を立て、各ステップにチェックボックスを付けて
{{ドキュメントルート}}/inception/units/units_plan.md ファイルに記載してください。

いずれかのステップで私の説明が必要な場合は、[Question] タグを付けて質問を追加し、
私が回答を記入するための空の [Answer] タグを作成してください。
[Question] には必ず番号を付けてください（例: [Question 1]）。
独自の判断や決定は行わないでください。

計画を作成したら、私のレビューと承認を求めてください。
承認後、同じ計画を1ステップずつ実行できます。
各ステップが完了したら、計画のチェックボックスに完了マークを付けてください。

あなたのタスク: {{ドキュメントルート}}/inception/user_stories.md のユーザーストーリーを参照してください。
```

| 変数 | 説明 | 例 |
|---|---|---|
| `{{ドキュメントルート}}` | ドキュメントの保存先ルートパス | `project-docs` |
