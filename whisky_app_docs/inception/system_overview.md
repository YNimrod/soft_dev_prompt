# システム概要（Phase 0 確定版）

## コンセプト

ウイスキーから始まる「スピリッツライフログアプリ」。
在庫管理が目的ではなく、飲んだ履歴・体験・感動を記録し、人と共有することが目的。

## 対象ユーザー

ウイスキーを趣味として日常的に飲んでいて、テイスティングノートを自分なりに記録したい中〜上級者。
（初心者層は当面の対象外。将来的な拡張対象として扱う）

## MVPの定義

MVP = このアプリを開発する上で最優先で実現する機能。
今回の要件定義を通じて、以下の3つに絞られた。

## MVPで実現したいこと（開発優先度順）

### 1. Collection（持ち物・経験の記録）

目的が異なる3つのリストを持つ。1本のボトルが複数リストに同時に属してよい（データの持ち方・排他性は設計フェーズで検討）。

- **飲んだことがあるウイスキーのリスト**
- **飲みたい・買いたいウイスキーのリスト**
- **持っているウイスキーのリスト**
  - この中で「未開栓／抜栓済み」をさらに区別したい（ユーザーが明示した要件）

缶ハイボールも対象に含める。開発優先度は他のボトルと同等に扱い、下げない
（現在話題性が高い分野であるため）。

### 2. Tasting（飲んだ記録・テイスティングノート）

飲んだ記録とテイスティングノートの記録・可視化。

### 3. SNS共有

自分の記録を見栄えよく外部SNSへ発信できる機能。
MVP内に含むが、3本柱の中では開発順序は最後になる。

## MVPに含むが範囲を絞るもの

- **Profile**: 自分の記録の統計可視化のみ。
  ポイント・スキルツリー・健康項目（性別・体重・年齢等）の入力は含めない。

## MVPでは扱わない・将来に回すもの

- **Map**（蒸留所・バー・イベント記録）: 開発優先度を下げ、Phase 2以降に延期
- **ヘルス機能**（アルコール代謝予測・カロリー算出・フードペアリング）
- **コミュニティ機能**（フレンド・公開範囲設定・好みベースのレコメンド）
- **ゲーミフィケーションの演出**（スキルツリー解放体験など）
  → 現在のターゲット（中〜上級者）に対しては優先度を下げてよいと判断

## 将来展望（MVP完成後）

- 焼酎・ジンなど他スピリッツへの拡張。
  Collection・Tastingの基本骨格（ボトル情報登録・テイスティング記録の仕組み）を
  応用展開する想定。アプリの大部分が完成した後の段階で着手する。

## まだ未決定だが要件定義をブロックしない論点

- 缶ハイボール・飲み方別のテイスティング軸の扱い
  （製法・原材料の違いをどう評価項目に反映するか）
  → テイスティング機能の詳細設計フェーズで再議論
