# ユーザーストーリー作成計画

## 概要

本ドキュメントは、「ウイスキーライフログアプリ（Whisky Life Log）」を開発するための契約となる、
明確に定義されたユーザーストーリーを作成するための作業計画です。各ステップは順番に実行し、
完了したらチェックボックスにマークを付けます。

作業を進めるにあたり、プロダクトマネージャーが独自に判断・決定することは行いません。
判断が必要な箇所はすべて [Question] タグで質問し、[Answer] を待ってから進めます。

---

## ステップ一覧

### ステップ 1: スコープと前提条件の確認
- [x] system_overview.md・goal_line.md・design_principles.md の内容を確認し、
      ユーザーストーリー作成の対象スコープを確定する

**確定したスコープ（B案・MVP全体7項目 ＋ MVP対象外4項目 ＝ 計11項目）:**

MVP対象（開発優先度: 高〜中）
1. ボトル登録（カメラ1本撮影／手動入力）と3リスト（持っている／飲んだことがある／飲みたい）への追加
2. テイスティング記録（評価項目入力＋ひとことコメント、一覧表示）
3. SNS共有（1件の記録を画像カード化して共有）
4. 缶ハイボールの登録・記録対応
5. 「持っている」リスト内の未開栓／抜栓済みの区別
6. 一枚の写真から複数本のボトルを検出する機能
7. Profile（統計可視化のみ）

MVP対象外・将来機能（ストーリーとして洗い出すが、開発優先度は低）
8. Map（蒸留所・バー・イベント記録）
9. ヘルス機能（アルコール代謝予測・カロリー算出・フードペアリング）
10. コミュニティ機能（フレンド・公開範囲設定・好みベースのレコメンド）
11. ゲーミフィケーションの演出（スキルツリー解放体験など）

※ 将来展望（他スピリッツ拡張）は対象外のまま（MVP完成後の構想のため、ストーリー化しない）

[Question 1] → [Answer 1]: B案（MVP全体）で確定
[Question 2] → [Answer 2]: MVP対象外4項目（Map・ヘルス・コミュニティ・ゲーミフィケーション）も追加で対象に含める
