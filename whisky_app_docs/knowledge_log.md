# ナレッジログ（whisky_app プロジェクト）

## Phase 0（要件の具体化・明確化）

- 対象ユーザーを「初心者〜上級者まで全員」のように広く設定すると、MVPのコアジャーニーが1本に絞れなくなる。早い段階で「最も価値を感じてほしい一人/一種類のユーザー」を明示的に問うことで、機能の絞り込みがスムーズに進んだ。
- 既存の検討資料（開発戦略メモ等）がある場合、要件定義の対話と資料の記載内容に時期的なズレ・矛盾が生じやすい（例: 「前倒しでPhase2」と書かれていた機能が、対話の中でMVP必須に確定するケース）。整合性レビューのタイミングで両者を突き合わせ、差分を申し送り事項として明示しておくと後工程で迷わない。
- 「データの持ち方・クラス設計」と「やりたいこと（要件）」を意識的に分離してもらうことで、要件定義フェーズが設計の細部に引き込まれずに進められた。ユーザーから「設計段階で煮詰めればいい」という明確な切り分けの意思表示があった。
