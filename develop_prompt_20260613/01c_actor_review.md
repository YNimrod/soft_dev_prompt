# Phase 1c: アクターレビュー

## このフェーズの目的

Phase 1b（整合性チェック）で内部品質・横断整合を確認した後、
**Phase 1 の ST2 で定義したアクター自身の視点**でユーザーストーリーを評価します。
「アクターが何をどのように行動するか」が明確に書かれているかを確認し、
行動の抜け漏れ・不自然なフロー・目的とのズレを早期に発見することが目的です。

---

## プロンプト

```
【作業ルール — 必ず守ること】
1. アクターを1人ずつ順番にレビューしてください。1アクター完了ごとに停止し、レビューを求めてください。
2. 各アクターで [Question] が発生した場合は番号を付け（例: [Question ACT-1]）、
   [Answer] を受け取るまで次のアクターに進まないでください。
3. 全アクターのレビューが完了したら「アクターレビューサマリー」を提示し、
   私の承認を求めて停止してください。
4. このフェーズのスコープ（レビューと問いの提示）を超えた修正は行わないでください。
   修正が必要な場合は「要修正リスト」として提示し、私の指示を待ってください。
5. ファイルを保存したら、保存完了を報告し、次の指示があるまで停止してください。

【Questionフォーマット — 必ずこの形式で記載すること】
[Question ACT-番号]
・問い   : 何を決める必要があるか（一文で）
・背景   : なぜこれが未決定なのか、なぜ問題になりうるか
・選択肢 : A案 / B案 /（不明な場合は「要調査」）
・影響   : 未決定のままだと〇〇フェーズの〇〇が進められない
[Answer ACT-番号]


━━━━━━━━━━━━━━━━━━━━━━━━━━━━
準備: アクター一覧の確認
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{{ドキュメントルート}}/inception/user_stories.md を参照し、
Phase 1 ST2 で定義されたアクター（ペルソナ）の一覧を提示してください。
あわせて {{ドキュメントルート}}/inception/phase1b_handover.md（Phase 1b の申し送り事項）が
存在する場合は参照し、申し送り事項に関連するアクターのレビューで考慮してください。
各アクターに関連するストーリー数を合わせて示してください。
提示後、私の確認を待ってからレビューを開始してください。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
レビュー: アクターごとに順番に実施
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
各アクターについて、以下の3観点でそのアクターに関連するストーリーをレビューしてください。

各アクターの出力フォーマット:
  ✅ 適切に定義されている点（箇条書き）
  ⚠️ 懸念点・問題（下記フォーマットの問題一覧で提示）
  ❓ 未回答の問い（[Question ACT-番号] 形式で提示）

【整合性確認 — 各アクターのレビュー提示前に必ず自己チェックすること】
各アクターのレビュー結果を提示する前に以下を確認し、「整合性確認レポート」として
レビュー結果と合わせて提示してください。

  ■ 今フェーズ内の確認
  - [ ] このアクターに関連するすべてのストーリーを確認したか（抜け漏れがないか）
  - [ ] 3観点（行動の自然さ・目的との対応・抜け漏れ）をすべて確認したか

  ■ 前フェーズ（Phase 1b）との整合
  - [ ] phase1b_handover.md の申し送り事項のうち、このアクターに関係するものを考慮したか
  - [ ] consistency_check.md の修正内容が、このアクターのストーリーに正しく反映されているか

【問題一覧のフォーマット】
| # | 観点 | 問題の概要 | 該当ストーリー | 影響度 | 推奨 |
|---|---|---|---|---|---|
| 1 | 行動の自然さ／目的との対応／抜け漏れ | （問題の概要） | ST-XX | 高／中／低 | 修正必須／修正推奨／任意 |

━━━━━━━━━━━━━━━━
観点 1: 行動の自然さ
━━━━━━━━━━━━━━━━
このアクターが実際に行うと想定される操作・行動として、各ストーリーの記述は自然か。

確認する内容:
  - ストーリーの操作手順がこのアクターの実際の行動パターンと一致しているか
  - 操作の前提条件（ログイン済み・権限あり等）が適切に設定されているか
  - アクターが実際には行わない操作がストーリーに含まれていないか

━━━━━━━━━━━━━━━━
観点 2: 目的との対応
━━━━━━━━━━━━━━━━
各ストーリーの「なぜ（目的・価値）」がこのアクターの本当の目的と合っているか。

確認する内容:
  - ストーリーに記述された目的・価値がこのアクターにとって意味があるか
  - 目的が曖昧・抽象的すぎて「誰のための機能か」が不明になっていないか
  - 複数のアクターに同じ目的が設定されている場合、それぞれの目的の違いが明確か

━━━━━━━━━━━━━━━━
観点 3: 抜け漏れ
━━━━━━━━━━━━━━━━
このアクターが必要とする操作・ストーリーで、現状の定義に含まれていないものがないか。

確認する内容:
  - このアクターの業務・行動フロー全体を考えたとき、明らかに欠けているストーリーがないか
  - 他のアクターのストーリーとの連携が必要な場面で、このアクター側のストーリーが欠けていないか
  - エラー時・例外時のアクターの行動がストーリーに含まれているか

【影響度の判定基準】
  高: Phase 3 以降（ユニット分割・PACT・ドメインモデル）で矛盾・手戻りを引き起こす
  中: Phase 3 以降で作業コストが上がるが致命的ではない
  低: 後工程への実害が少ない（読みやすさの問題など）

【推奨の基準】
  修正必須: 影響度「高」。このまま進むと後工程で手戻りが発生する
  修正推奨: 影響度「中」。修正しておくと後工程がスムーズになる
  任意    : 影響度「低」。修正しなくても後工程への実害は少ない

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
全アクター完了後: アクターレビューサマリーの提示
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
すべてのアクターのレビューが完了したら、以下の形式で総合サマリーを提示してください。

## アクターレビューサマリー

### 次フェーズへの引き継ぎ事項
（Phase 1d 以降で考慮すべきアクター行動上の前提・制約を列挙）

【中間成果物の保存】
アクターレビューサマリーが確定したら、以下のファイルを必ず保存してください。
- {{ドキュメントルート}}/inception/actor_review.md       … アクターレビュー結果 ＋ 総合サマリー

【ナレッジの記録 — フェーズ完了時に必ず実施すること】
完了後、このフェーズで得られた知見を以下に追記してください。
- {{ドキュメントルート}}/knowledge_log.md
  追記内容の例:
  - 「〇〇アクターの行動フローで△△の抜け漏れが発見された（再利用可能なチェックパターン）」

```
