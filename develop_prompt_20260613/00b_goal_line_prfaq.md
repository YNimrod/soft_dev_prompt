# Phase 0b: ゴールライン定義（PRFAQ）

## このフェーズの目的

Phase 0（`00_grill_me.md`）で固めた要件をもとに、
「最低限ここまでやれば成功と言えるライン（ゴールライン）」を決めます。

ゴールラインとは「最高到達点（将来的な理想形）」ではなく、
「これを満たせば成功と言える最低到達点」のことです。
これを先に決めておくことで、Phase 1以降での過剰仕様化・スコープ膨張を防ぎます。

複数の機能を持つプロダクトの場合、機能ごとに個別の完成度でゴールラインを
決めるのではなく、「ユーザーが最初に価値を体験するまでの一本道（コアジャーニー）」を
基準に、各機能から必要最小限を切り出す形で決めます。

---

## プロンプト

```
あなたの役割: あなたは経験豊富なプロダクトマネージャーです。
PRFAQ（プレスリリース＋FAQ）の手法を用いて、プロダクトの最低到達点（ゴールライン）を
明確化する任務を担っています。

これから、Phase 0で固めた要件（system_overview.md）をもとに、
「最低限ここまでやれば成功と言えるライン（ゴールライン）」を決めます。

1. まず、完成後に発表する想定のプレスリリース文（PRFAQ）を1〜2段落で下書きしてください。
   - 誰のためのプロダクトか／どんな課題を解決するか／使うとどう変わるか、を簡潔に

2. その下書きから、「ユーザーが最初に価値を体験するまでの一本道（コアジャーニー）」を
   一連のステップとして書き出してください。
   （例: 記録する → 分析結果を見る → 共有する）

3. コアジャーニーの各ステップについて、それを通過するために
   各機能が持つべき「最小限の実装」を1つずつ定義してください。
   ここでの目的は機能を完成させることではなく、ジャーニーを成立させることです。

4. ゴールラインに含めないもの（後回しでよい機能・パターン・例外処理）も明示してください。

5. 上記1〜4の内容について、以下の観点でレビューしてください。
   - 各機能が「何%完成」という進捗表現になっていないか（機能単位の完成度ではなく、
     ジャーニー単位の必須/不要で語られているか）
   - 全ての機能が、同じ1本のコアジャーニーに紐づいているか
     （別々のシナリオがバラバラに存在していないか）
   - コアジャーニーを実際に最初から最後まで一人のユーザーが体験できる順序になっているか
   - 「ゴールラインに含めないもの」が、ジャーニーの成立を妨げないか
   - PRFAQの内容とコアジャーニー・ゴールラインの内容に矛盾がないか

   いずれかの観点で不十分な点が見つかった場合は、その内容を私に提示し、
   私と一緒に3または4の内容を修正してください。
   すべての観点をクリアするまで、このレビューと修正を繰り返してください。

ゴールラインが固まったら、以下のファイルに保存してください。
- {{ドキュメントルート}}/inception/goal_line.md … PRFAQ・コアジャーニー・ゴールライン・対象外項目

【ナレッジの記録】
保存後、このフェーズで得られた知見を以下に追記してください。
- {{ドキュメントルート}}/knowledge_log.md
  追記内容の例:
  - 「ゴールライン設定で機能ごとの分散になりやすかったケースと、その修正パターン」
```

## 使い方

`00_grill_me.md` で `system_overview.md` を確定させた後、このプロンプトを送信してください。
PRFAQ・コアジャーニー・ゴールラインの下書きが提示されたら、内容を確認し、
レビュー観点での指摘があれば一緒に修正してください。
