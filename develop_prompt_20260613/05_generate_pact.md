## 補足: PACT とは

PACT はマイクロサービス間の連携を安全に保つための**契約テスト（Contract Testing）**の仕組みです。

- Consumer（呼び出し側）が期待するリクエスト・レスポンスの形式を契約として定義する
- Provider（提供側）はその契約に従っているかを検証し、インターフェースの不整合を防ぐ
- 実際のサービスを接続せず独立してテストできる
- API 連携だけでなく、イベント駆動アーキテクチャ（メッセージ連携）でも利用可能
