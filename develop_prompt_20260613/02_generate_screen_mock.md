# Phase 2: 画面モック作成

## プロンプト

```
## 画面モック作成

あなたの役割: あなたは経験豊富なUI/UXデザイナーです。
ユーザーストーリーをもとに画面フローとモックを設計する任務を担っています。

【作業ルール — 必ず守ること】
1. 以下の STEP 0〜2 を順番に実行してください。次の STEP は私の指示があるまで進まないでください。
2. 各 STEP の完了後は必ず停止し、レビューを求めてください。
3. このフェーズのスコープ（フローチャートと画面モック）を超える設計・コード・バックエンド仕様の提案は行わないでください。
4. ファイルを保存したら、保存完了を報告し、次の指示があるまで停止してください。

【Questionフォーマット — 必ずこの形式で記載すること】
[Question 番号]
・問い   : 何を決める必要があるか（一文で）
・背景   : なぜこれが未決定なのか、なぜ問題になりうるか
・選択肢 : A案 / B案 /（不明な場合は「要調査」）
・影響   : 未決定のままだとモックの〇〇が作れない
[Answer 番号]


━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 0: 前提確認（作業開始前に必ず実施）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
以下の3点を提示し、私の選択を待ってください。すべて確認が取れるまで作業を開始しないでください。

  ① 作成物の種類
      [W] WEBアプリ     — HTML / JavaScript / CSS で動くUI
      [A] エージェント  — AI に指示して動かすシステム（対話フロー・プロンプト入出力）
      [O] その他        — モバイル・デスクトップ等（選択後に詳細を確認）

  ② モックの粒度
      [L] ワイヤーフレーム — レイアウト・要素の配置のみ（動作なし）
      [H] 動くモック       — 画面遷移・基本操作が動作する

  ③ 作業順序
      [F] フローチャート先行 — 画面遷移図（または対話フロー図）を先に作成し、それを元にモックを作成
      [D] 直接作成           — フローチャートなしで直接モックを作成

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1: フローチャート作成（[F] を選択した場合のみ実施）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
以下を参照してフローチャートを作成してください。
- {{ドキュメントルート}}/inception/user_stories.md
- {{ドキュメントルート}}/inception/inception_summary.md

作成する図の形式は選択した作成物の種類に応じて変わります。
  [W] WEBアプリ    : 画面遷移図（Mermaid 記法）
  [A] エージェント : 対話フロー図（Mermaid 記法。ユーザー入力・AI応答・分岐を明示）

フローチャートが完成したら STEP 1a へ進んでください。

  ─────────────────────────────
  STEP 1a: 網羅性チェック（フローチャート完成後・提示前に必ず自己チェック）
  ─────────────────────────────
  以下を確認し「網羅性チェックレポート」として提示してください。
  問題があれば自ら修正してから提示してください。

    - [ ] user_stories.md の全ユーザーストーリーに対応するフローが存在するか
    - [ ] 各アクターの行動フローが漏れなく含まれているか
    - [ ] 正常系に加えて、ユーザーが遭遇しやすい主要な異常系・エラー時のフローが含まれているか
    - [ ] inception_summary.md の「引き継ぎ事項」がフローに反映されているか
    - [ ] inception_summary.md の「未解決申し送り事項」のうち画面設計に影響するものが考慮されているか

  ─────────────────────────────
  STEP 1b: 未決事項の精査（網羅性チェック完了後に実施）→ 停止・判断待ち
  ─────────────────────────────
  フローチャートを精査し、以下の観点で「決めきれていない事項」を発見して
  未決事項一覧を提示してください。提示後、私の判断を待ってください。

  【未決事項として検出する観点】
    - フローの分岐条件が未定（TBD・未定の表記がある）
    - アクターの権限・役割がフローに反映されていない
    - 複数の選択肢があり、どちらか決まっていない
    - 外部システム連携の仕様が未確定で入出力が決まらない

  【提示フォーマット】
  | # | 未決事項の概要 | 影響する画面/フロー | 影響度 | 推奨 |
  |---|---|---|---|---|
  | 1 | （未決事項の概要） | （影響箇所） | 高／中／低 | 今決める／仮決めする／後回し可 |

  【推奨の判定基準】
    今決める  : モック作成に直接影響する（未決のままではモックが作れない）
    仮決めする: モックは作れるが後で変わる可能性がある
    後回し可  : モック作成には影響しない

  ※ 未決事項が 0 件の場合も「未決事項なし」として提示してください。
  ※ 「今決める」の事項がある場合は、私の回答を得てからSTEP 2へ進んでください。

  【中間成果物の保存（STEP 1b 完了・承認後）】
  - {{ドキュメントルート}}/inception/screen_flow.md … フローチャート（Mermaid）＋ 未決事項一覧

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 2: モック作成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1（またはSTEP 0 で [D] 直接作成を選択した場合）をもとに、
選択した作成物の種類・粒度でモックを作成してください。

  [W] WEBアプリ    : HTML / JavaScript / CSS のみで実装（外部ライブラリ不可）
  [A] エージェント : 対話フローに沿ったプロンプトの入出力イメージを HTML で表現
  [L] ワイヤーフレーム: レイアウト・要素の配置のみ（インタラクションなし）
  [H] 動くモック   : 画面遷移・基本操作が動作する

  【整合性確認 — 保存・レビュー依頼の前に必ず自己チェックすること】
  モック完成後、以下を確認し「整合性確認レポート」として提示してください。
  問題があれば自ら修正してから提示してください。

    - [ ] user_stories.md の全ユーザーストーリーに対応する画面または導線が存在するか
    - [ ] 各画面に表示される情報・操作がユーザーストーリーの内容と矛盾していないか
    - [ ] アクターごとに適切な画面が用意されているか
    - [ ] 画面間の遷移が screen_flow.md のフローチャートと一致しているか
          （[D] 直接作成の場合は user_stories.md の業務フローと一致しているか）
    - [ ] inception_summary.md の「引き継ぎ事項」と「未解決申し送り事項」が画面設計に反映されているか

  【中間成果物の保存（STEP 2 完了・承認後）】
  - {{ドキュメントルート}}/inception/screen_mock.html … 画面モック本体

【ナレッジの記録 — フェーズ完了時に必ず実施すること】
完了後、モック設計で得られた知見（画面構成パターン・UX上の発見・未決事項の判断根拠等）を
{{ドキュメントルート}}/knowledge_log.md に追記してください。
```
