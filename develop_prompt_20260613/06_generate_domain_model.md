# Phase 6: ドメインモデリング

## プロンプト

```
## ドメインモデリング

あなたの役割: あなたは経験豊富なソフトウェアエンジニアです。
タスクセクションで参照されているすべてのユーザーストーリーを実装するための
ドメインモデルを設計する任務を担っています。

このモデルには、ユーザーストーリーのビジネスロジックを実装するための
すべてのコンポーネント・属性・振る舞い、およびコンポーネント間の相互作用が含まれます。

以下は含めないでください:
- アーキテクチャコンポーネント
- コード

コンポーネントモデルは
{{ドキュメントルート}}/construction/<ユニット名>/docs フォルダの
個別の md ファイルに記載してください。

作業の計画を立て、各ステップにチェックボックスを付けて
{{ドキュメントルート}}/construction/<ユニット名>/domain_model_plan.md ファイルに記載してください。

いずれかのステップで私の説明が必要な場合は、[Question] タグを付けて質問を追加し、
私が回答を記入するための空の [Answer] タグを作成してください。
[Question] には必ず番号を付けてください（例: [Question 1]）。
独自の判断や決定は行わないでください。

計画を作成したら、私のレビューと承認を求めてください。
承認後、同じ計画を1ステップずつ実行できます。
各ステップが完了したら、計画のチェックボックスに完了マークを付けてください。

あなたのタスク: {{ドキュメントルート}}/inception/units/{{対象ユニットのmdファイル名}} を参照してください。
```

---

## 使用する変数

| 変数 | 説明 | 例 |
|---|---|---|
| `{{ドキュメントルート}}` | ドキュメントの保存先ルートパス | `project-docs` |
| `{{対象ユニットのmdファイル名}}` | ドメインモデリング対象ファイル | `unit_01_browsing.md` |
| `<ユニット名>` | 出力先フォルダ名（対象ユニットに合わせて変更） | `unit_01_browsing` |

---

## 運用メモ

> ✅ **このフェーズはユニットごとに繰り返し実行する（ユニット数分）**  
> ✅ **実装順序: ① ドメインモデリング → ② ビジネスロジック実装 → ③ クラウドリソース選定 → ④ デプロイ**
